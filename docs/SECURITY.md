# নিরাপত্তা মডেল (Security Model)

## এনক্রিপশন কীভাবে কাজ করে

1. **Key Derivation:** মাস্টার পাসওয়ার্ড থেকে PBKDF2 (SHA-256, 250,000 iterations) দিয়ে একটি 256-বিট AES key তৈরি হয়। একটি র‍্যান্ডম 16-byte salt প্রতিটি ভল্টের জন্য আলাদাভাবে জেনারেট হয়।
2. **Verification:** মাস্টার পাসওয়ার্ড সঠিক কিনা যাচাই করতে একটি নির্দিষ্ট verifier স্ট্রিং এনক্রিপ্ট করে রাখা হয় — আসল ডেটা কখনো এই উদ্দেশ্যে ব্যবহার হয় না।
3. **Storage Encryption:** সব এন্ট্রি (site, username, password, notes) একসাথে AES-256-GCM দিয়ে এনক্রিপ্ট করে ব্রাউজারের `localStorage`-এ রাখা হয়। প্রতিটি সেভে নতুন random IV ব্যবহার হয়।
4. **Key Lifetime:** Derive করা key শুধু মেমোরিতে (RAM) থাকে, কখনো ডিস্কে সংরক্ষণ হয় না। ভল্ট লক করলে বা ট্যাব বন্ধ করলে key মুছে যায়।

## যা এই মডেল প্রতিরোধ করে

- ব্রাউজার storage ফাইল সরাসরি দেখলে কেউ প্লেইনটেক্সট পাসওয়ার্ড পাবে না
- মাস্টার পাসওয়ার্ড ছাড়া এনক্রিপ্টেড ব্যাকআপ ফাইল ডিক্রিপ্ট করা যাবে না

## যা এই মডেল প্রতিরোধ করে না

- **Keylogger / malware:** যদি আপনার ডিভাইসে ইতিমধ্যে ম্যালওয়্যার থাকে যা টাইপ করা জিনিস রেকর্ড করে, কোনো ক্লায়েন্ট-সাইড এনক্রিপশনই সাহায্য করবে না।
- **দুর্বল মাস্টার পাসওয়ার্ড:** ছোট বা সহজে অনুমানযোগ্য পাসওয়ার্ড brute-force আক্রমণের ঝুঁকিতে থাকে। কমপক্ষে ১২-১৬ অক্ষরের এলোমেলো/passphrase-স্টাইল পাসওয়ার্ড ব্যবহার করুন।
- **ব্রাউজার এক্সটেনশন:** সন্দেহজনক ব্রাউজার এক্সটেনশন পেজের DOM/ক্লিপবোর্ড পড়তে পারে।
- **ডিভাইস চুরি (আনলকড অবস্থায়):** ভল্ট আনলক থাকা অবস্থায় ডিভাইস হাতছাড়া হলে ডেটা এক্সপোজড।

## দায়িত্বশীল প্রকাশ (Responsible Disclosure)

নিরাপত্তা দুর্বলতা খুঁজে পেলে অনুগ্রহ করে সরাসরি রিপোজিটরির মেইনটেইনারকে জানান, পাবলিক Issue হিসেবে নয়। যুক্তিসঙ্গত সময়ের মধ্যে সমাধানের চেষ্টা করা হবে।

## এটি একটি অডিটেড প্রোডাক্ট নয়

এই প্রজেক্টটি ব্যক্তিগত/শিক্ষামূলক উদ্দেশ্যে তৈরি। এটি কোনো প্রফেশনাল সিকিউরিটি অডিট এর মধ্য দিয়ে যায়নি। অত্যন্ত সংবেদনশীল ডেটার জন্য প্রতিষ্ঠিত, স্বাধীনভাবে অডিটেড পাসওয়ার্ড ম্যানেজার (যেমন Bitwarden, KeePass) ব্যবহার বিবেচনা করুন।
